When a program consumes another tool's output programmatically, drive that tool in its machine-readable or unambiguous mode — a delimited, structured, or escaping-free format meant for scripts — rather than parsing the output intended for human display. Human-facing output applies quoting, escaping, truncation, localization, alignment, or separators that a literal parse silently mishandles, dropping or corrupting records (a name with a space or a non-ASCII byte that gets quoted, a wrapped column, a localized number). The machine format exists precisely so a consumer need not reverse-engineer the presentation layer; choosing it eliminates a whole class of edge-case parsing bugs instead of patching them one at a time, and its own quirks (multi-field records, a trailing delimiter) are then handled deliberately. Flag code that parses a tool's default or pretty-printed output where a documented machine-readable mode is available, especially where the human format quotes or escapes values the parse then reads verbatim.
