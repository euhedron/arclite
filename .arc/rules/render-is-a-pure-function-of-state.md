A function that produces output from state — a render, a view, a serializer, a projection — must be a pure function of that state: it reads state and returns output, with no mutation, I/O, or environment probing of its own. Push side effects and fallible probes (filesystem, clock, network, environment, loading external data) onto the one update path that owns state changes, or into a one-time setup step, and store their results in state for the pure function to read. This keeps the output deterministic and verifiable in isolation — state in, output out — and concentrates mutation in one auditable place instead of scattering it through the code that produces output. Flag a render/view/serialize routine that mutates shared state, reads the filesystem/clock/network/environment inline, or otherwise carries a side effect, and hoist the effect onto the mutation path or precompute it into state.
