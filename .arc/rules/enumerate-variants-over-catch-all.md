When branching over a closed set of variants — an enum, a tagged union, a sealed type — name each case explicitly wherever a future member could need handling of its own, rather than absorbing the remainder into a wildcard or default arm. An explicit enumeration turns "a variant was added" into an exhaustiveness error at every site that must decide about it, so the compiler (or checker) walks the author to each decision; a catch-all converts that compile-time prompt into a silent runtime misbehavior, because the new member falls through to logic chosen before it existed. A catch-all is legitimate only where the absorbed cases genuinely share behavior that remains correct for any member added later — dispatch-irrelevant defaults, not unconsidered ones. Flag a wildcard/default arm that stands in for real, individually-meaningful variants — especially one whose branch performs an action (launch, write, spend) a future variant would need to opt out of.
