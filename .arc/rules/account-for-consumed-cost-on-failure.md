An operation that consumes a metered resource as it runs — money, tokens, quota, rate-limited calls — must still capture and record what it consumed when it fails or its result is discarded: only the unusable result is dropped, never the cost. A failure path that returns before the accounting step a success would reach makes totals understate true spend, and hides exactly the runaway or partial-failure cost the accounting exists to surface. Route failures through the same recording path as successes, distinguished by an explicit error marker; reconstruct consumption from the failure payload when the normal usage report is voided on error; and fold a concurrent fan-out's failed members' spend into the total alongside the survivors'. Flag an error or early-abort path that discards already-incurred consumption along with the failed result, or an aggregate that sums only successful units' cost.
