Code that switches a shared external resource into a non-default mode for the duration of an operation — terminal raw mode, an alternate or inline screen, a global hook, an acquired lock, a process-wide setting — must restore it on every exit path: the normal return, early returns, errors, and panics alike. A resource left in its modified state corrupts whatever uses it next — a shell stranded in raw mode, a lock never released, a hook that never fires again — and that corruption outlives the operation that caused it. Make the restoration unskippable: a scope guard, a destructor, or an unconditional teardown that runs even when the owning operation failed — never a cleanup statement reachable only on the success path. Teardown ordering is part of the contract: a step that depends on the modified mode must run before the step that exits it. While the resource is held exclusively, also forbid the ordinary side channels that would corrupt it. Flag an operation that enters such a mode but restores it only on normal completion, skips restoration on an error or panic path, or exits the modes in an order that still uses the resource after its mode is gone.
