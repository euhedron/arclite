A process that captures more than one of a child process's output streams — stdout and stderr, or any set of pipes — must drain them concurrently (separate threads, or async/select), never one stream to completion before reading the next. A child that fills the OS pipe buffer of a stream nobody is currently reading blocks on that write; if the parent is meanwhile blocked reading a different stream that will only advance once the child unblocks, the two deadlock. Drain each captured stream on its own thread (or via async) and join after the child exits. Flag a subprocess driver that captures multiple output pipes but consumes them sequentially — reading one fully and only then the other.
