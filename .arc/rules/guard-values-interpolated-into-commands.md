A value spliced into a constructed command line, shell invocation, config directive, query, or any other text a downstream parser interprets must be validated or escaped for that destination's grammar at the point of interpolation, so it cannot break out of its slot and inject adjacent directives — even when it came from a trusted-seeming source such as an environment variable or a prior call. A quote, newline, separator, or comment character embedded in the value can terminate the surrounding quoting and append options or statements the author never intended. Constrain the value to its expected shape where it is embedded — reject the characters significant to that grammar, or use a parameterizing or escaping API — rather than trusting it to be well-formed. This is the directive-injection counterpart to validating path components: a path guard stops traversal, this stops injection into a command or config grammar. Flag an externally-sourced value concatenated into a command, config line, or query with no shape check or escaping at the interpolation site.
