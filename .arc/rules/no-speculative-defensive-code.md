Flag code that handles a condition with no evidence of occurring — a speculative option, default, fallback, or branch for a case never observed and not part of any documented contract — and, equally, code that treats an effectively-infallible operation as though it might fail. Match the handling to the case: a genuinely fallible operation (external I/O, parsing external output, the environment) must surface its real failure honestly, propagated or recovered deliberately; a condition that cannot occur given the code's own invariants should be asserted, failing loudly only if that invariant is ever broken. Two opposite moves both hide reality — fabricating a plausible substitute value that masks the case, and wrapping an infallible call in recoverable-error plumbing as though it were a normal failure mode (mistrust of a guarantee the code already holds). Erroring or asserting on a should-never-happen condition is how it would reveal itself; speculative handling in either direction is the real cost.
