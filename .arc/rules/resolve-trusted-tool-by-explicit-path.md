When code depends on a specific implementation of an external executable — for its security properties, its platform integration (a system TLS or certificate stack), or its exact behavior — invoke it by a trusted, canonical path rather than a bare name resolved through the environment's search path, which an earlier shadowing install could silently substitute (a correctness hazard and a known hijacking vector). Probe the canonical location and fall back to a search-path lookup only when that location is confirmed absent, never when the probe is merely inconclusive. Where the tool's identity does not matter, a search-path lookup is fine; this targets the case where a particular build is a real precondition. Flag a security- or platform-sensitive subprocess spawned by bare name when a specific implementation is required.
