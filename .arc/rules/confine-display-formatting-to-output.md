A transformation applied purely to make a value readable for a person — abbreviating a path prefix, truncating, prettifying, localizing, or re-casing — belongs only at the point the value is shown, and must never be applied to a value that is then stored, used as a key or for lookup, re-parsed, compared, or embedded in an error or diagnostic message. Those consumers need the exact, canonical form: a display-abbreviated path can no longer be opened, a prettified record stops matching its stored form, and an error naming a mangled value misdirects whoever must act on it. Keep one canonical representation flowing through storage, lookups, comparisons, and error text, and derive the human-facing string from it only at the output boundary. Flag a display-formatting, abbreviation, or truncation helper whose result flows into stored state, a later lookup or re-parse, or an error message, rather than only into rendered output.
