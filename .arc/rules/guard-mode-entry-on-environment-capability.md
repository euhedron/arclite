Before entering a mode that assumes an environment capability — raw or interactive terminal mode, an alternate-screen takeover, TTY-only or colored output, any path that needs a runtime facility to be present — probe for that capability first and refuse cleanly, with a clear message, when it is absent, rather than entering the mode regardless and then hanging, corrupting a stream, or emitting garbage. This probe is a legitimate environment check, not speculative defensiveness: the capability is a real precondition of the mode, so proceeding without it is the actual failure, not a hypothetical one. The refusal belongs at the boundary, before any state is changed, so the caller gets an honest error rather than a wedged or corrupted session. Flag a routine that unconditionally enters such a mode without first verifying the precondition it depends on.
