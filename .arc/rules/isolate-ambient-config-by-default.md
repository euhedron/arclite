When invoking an external tool whose behavior ambient configuration can silently steer — user dotfiles, auto-loaded project instructions or memory, inherited global settings — and the run reports its parameters as the authoritative account of what shaped it, isolate the invocation by default: set every behavior-shaping factor explicitly, suppress the ambient sources, and disclose the isolation state in the report. The point is standardized inputs and an honest report, not deterministic output — the tool itself may be nondeterministic; what isolation guarantees is that the same command means the same thing across machines and users, and that no unstated ambient source falsifies the claim that the declared parameters were the whole story. Loading the ambient context is legitimate only as an explicit, named, reported opt-in. Flag a subprocess invocation that inherits environment-level config or auto-loaded context by default while presenting its parameter report as complete.
