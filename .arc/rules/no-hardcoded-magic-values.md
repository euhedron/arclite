Flag magic values hardcoded into logic — numeric thresholds, empirically-derived constants, model/version/window sizes, limits, embedded URLs or paths — especially those that drift as the world or dependencies change. Prefer values that are configurable, derived from an authoritative source at runtime, or at minimum named and documented with their rationale (and, for empirical values, a refresh cadence). A bare unexplained literal, or an assumed context-window size, silently goes stale and misleads. A single value that is already named, single-sourced, and documented with its rationale (and, where it can drift, an update note) satisfies this — the target is the bare or scattered literal, not the named constant that remedies it.
