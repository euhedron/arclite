A non-essential side effect — logging, telemetry, a progress or status marker, any write whose only purpose is to observe or record the primary work — must be best-effort: if it fails, surface a warning so the failure is visible, then let the primary operation proceed unaffected. The success of real work must never hinge on the bookkeeping that merely observes it. This is the deliberate, disclosed counterpart to swallowing a genuine failure, which remains forbidden: the line is that the degraded operation is genuinely non-essential *and* its failure is announced, not hidden. Flag both directions — an auxiliary, observe-only write whose error aborts real work, and a best-effort write that fails completely silently with no warning.
