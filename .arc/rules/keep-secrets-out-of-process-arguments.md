A secret used at runtime — a credential, token, API key, or password — must never be passed to a child process as a command-line argument. A process's argument vector is readable by other users on the host through the operating system's process table, and arguments routinely persist in shell history and logs, so a secret placed on argv is exposed the moment the process is spawned, however securely it was obtained. Hand it to the child through a channel the process table does not expose — the child's standard input, an environment variable scoped to that child, or a credential file or store it reads directly — preferring stdin or an environment variable over a temporary file, whose permissions vary by platform and which can outlive the call. This is the runtime counterpart to keeping secrets out of committed source: a credential can be sourced perfectly safely and still leak through the way a subprocess is invoked. Flag a credential, token, or key interpolated into a command's argument vector where a channel the process table does not expose would serve.
