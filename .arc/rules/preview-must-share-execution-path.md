When a tool offers a preview, dry run, estimate, or plan of an action before performing it, that preview must be produced by the same code path as the real action — one construction both build on — so it cannot drift from what actually runs. A preview assembled by parallel, re-derived logic can silently diverge (different flags, a narrower scope, a stale estimate), turning a feature meant to build trust into a source of false confidence — most dangerously when the preview is what gates a costly or irreversible step. Flag a dry run, estimate, or plan whose command, parameters, or computed effects are built separately from the execution path rather than sharing one definition; prefer deriving the preview from the real path — capturing the actual tool's own preview output — over reconstructing what you assume it will do.
