When an authenticated request may be redirected to a different host — a download endpoint that bounces to signed object storage, an API that redirects to a CDN — the credential it carries (an authorization header, a basic-auth pair, a token) must not be forwarded onto the redirected request. The new host has no need for it, and a redirect to an attacker-influenced location would leak it across a trust boundary. When following redirects with a credential attached, confirm the client strips it on a cross-origin hop — most do by default, some do not — and never blanket-enable sending the credential on every redirect. Flag a redirect-following authenticated fetch that forwards the credential across hosts, or one that depends on cross-host credential-stripping without that hop being acknowledged.
