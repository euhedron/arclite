A rule must state a general principle that holds across repositories, languages, and contexts — not wording bound to one codebase's particulars. Flag a rule that pins itself to specific identifiers, symbol or file names, flags, repository names, concrete code excerpts, or enumerations of the system's current state (which components do what, present configuration values, the way a hook or setting happens to be wired today). Such specifics are unnecessary when the principle is clearly stated, and they go stale as the code moves. Naming general categories and language-agnostic concepts is fine; binding the rule to a particular instance is the defect.
